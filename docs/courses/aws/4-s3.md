# Lesson 4 — S3: Object Storage

> **Module 2 · Core Compute & Storage · Lesson 4 of 10** &nbsp;•&nbsp; **SAA-C03 Domain 1 & 3:** Secure storage, elastic/secure content delivery
> 📌 *Authoritative source:* [S3 User Guide](https://docs.aws.amazon.com/AmazonS3/latest/userguide/) · [S3 CLI](https://awscli.amazonaws.com/v2/documentation/api/latest/reference/s3/index.html) — verified against AWS docs.

S3 is the most-tested service on SAA. Know **durability/availability, storage classes, versioning, encryption, lifecycle, and access control** precisely.

## 1. Durability, availability, scalability

- **11 nines (99.999999999%) durability** — designed to lose ~1 object per 10B years.
- **99.99% availability** (Standard) — distinct from durability.
- **Unlimited** storage, objects 0 B–5 TB, flat namespace (no real folders).

## 2. Storage classes (exam must-know)

| Class | Min dur | Use |
|-------|---------|-----|
| **Standard** | — | frequent, millisecond access |
| **Intelligent-Tiering** | 30d (small monthly fee) | unknown/variable; auto-moves tiers |
| **Standard-IA** | 30d, 128 KB min | infrequent, fast retrieval |
| **One Zone-IA** | 30d, 128 KB | infrequent, non-critical, one AZ |
| **Glacier Instant** | 90d | archive, ms retrieval |
| **Glacier Flexible** | 90d | min/s–hours retrieval |
| **Glacier Deep Archive** | 180d | 12h retrieval, cheapest |
| **Reduced Redundancy** | — | deprecated, avoid |

> 🧠 **Exam tip:** "Unknown access pattern, don't want to manage tiers" → **Intelligent-Tiering**. "Compliance archive, rarely read" → **Glacier Deep Archive**.

## 3. Versioning & lifecycle

- **Versioning** (once on, can only suspend) protects against accidental delete/overwrite; deleted objects become a *delete marker*.
- **MFA Delete** adds an extra layer for version changes.
- **Lifecycle rules** transition objects between classes or expire them (cost optimization — a top SAA theme).

<ExampleBox title="Lifecycle: IA at 30d, Glacier at 90d" lang="json">

```json
{
  "Rules": [{
    "Id": "tier", "Status": "Enabled", "Prefix": "",
    "Transitions": [
      {"Days": 30, "StorageClass": "STANDARD_IA"},
      {"Days": 90, "StorageClass": "GLACIER"}
    ],
    "Expiration": {"Days": 365}
  }]
}
```

</ExampleBox>

## 4. Encryption (at rest)

- **SSE-S3** (`AES256`) — AWS-managed keys, simplest.
- **SSE-KMS** — customer-managed KMS keys, audit + key policy + rotation.
- **SSE-C** — you supply/own the key per request.
- **Bucket keys (SSE-KMS)** reduce KMS cost by caching.
- Enforce via **bucket policy** `Deny` when `x-amz-server-side-encryption` is absent.

<ExampleBox title="Enforce SSE" lang="bash">

```bash
aws s3api put-bucket-encryption --bucket my-bucket-2026 \
  --server-side-encryption-configuration '{"Rules":[{"ApplyServerSideEncryptionByDefault":{"SSEAlgorithm":"aws:kms","KMSMasterKeyID":"<key-arn>"}}]}'
```

</ExampleBox>

## 5. Access control & blocking public access

- **Bucket Policy** (resource-based, JSON, supports `aws:SourceIp`, `aws:PrincipalOrgID`).
- **ACL** (legacy; AWS now recommends disabling). **S3 Block Public Access** should be ON unless intentionally public (static site).
- **VPC Endpoint (Gateway)** lets S3 be reached privately without internet — keeps traffic on the AWS network.

## 6. Static hosting + CloudFront (see capstone)

`aws s3 website` + a CloudFront distribution (covered Lesson 10). For private content, use **CloudFront Origin Access Control (OAC)** so only the CDN can read the bucket.

## 7. Exercises

<ExerciseBox title="Storage class + lifecycle" difficulty="Easy">
Create a bucket, `aws s3 cp` a file with `--storage-class STANDARD_IA`, then `aws s3 ls --recursive` to confirm. Put a lifecycle rule expiring objects at 365d.
</ExerciseBox>

<ExerciseBox title="Encryption enforcement" difficulty="Medium">
Enable SSE-KMS on a bucket and add a bucket policy that **denies** `s3:PutObject` unless `x-amz-server-side-encryption` is `aws:kms`. Attempt an unencrypted upload and observe the Deny.
</ExerciseBox>
