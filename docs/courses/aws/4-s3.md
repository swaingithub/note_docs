# Lesson 4 — S3: Object Storage

S3 (Simple Storage Service) is infinitely scalable object storage. You store **objects** (files) in **buckets**. It backs everything from static websites to data lakes.

## Buckets & objects

- **Bucket** — a container with a globally unique name.
- **Object** — a file + metadata, addressed by key.
- **Key** — the object's name/path inside the bucket.
- **Region** — buckets live in one region.

## Create and use a bucket

<ExampleBox title="Bucket operations" lang="bash">

```bash
aws s3 mb s3://my-unique-bucket-2026
echo "hello" > file.txt
aws s3 cp file.txt s3://my-unique-bucket-2026/file.txt
aws s3 ls s3://my-unique-bucket-2026
aws s3 cp s3://my-unique-bucket-2026/file.txt ./downloaded.txt
```

</ExampleBox>

## Sync (powerful)

<ExampleBox title="Sync a folder" lang="bash">

```bash
aws s3 sync ./site s3://my-unique-bucket-2026 --delete
```

</ExampleBox>

`--delete` removes objects in the bucket that are not in the local folder.

## Static website hosting

<ExampleBox title="Enable static site" lang="bash">

```bash
aws s3 website s3://my-unique-bucket-2026 \
  --index-document index.html --error-document error.html
# then make objects public (or use a bucket policy / CloudFront — see Lesson 10)
```

</ExampleBox>

## Storage classes

| Class | Use |
|-------|-----|
| Standard | Frequently accessed |
| Intelligent-Tiering | Unknown/unpredictable access |
| Standard-IA | Infrequent, fast retrieval |
| Glacier | Archive (minutes–hours retrieval) |

<ExerciseBox title="Bucket lifecycle" difficulty="Easy">
Create a bucket, upload three files, list them, then delete one with `aws s3 rm`. Confirm the listing shows two remaining.
</ExerciseBox>

<ExerciseBox title="Sync + classes" difficulty="Medium">
Sync a local folder to a bucket, set the objects to `STANDARD_IA` with `--storage-class`, then `aws s3 ls --recursive` and confirm the objects are present.
</ExerciseBox>
