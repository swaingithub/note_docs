# Lesson 3 — EC2: Compute

> **Module 2 · Core Compute & Storage · Lesson 3 of 10** &nbsp;•&nbsp; **SAA-C03 Domain 1 & 2:** Secure/Resilient compute, high-performance/elastic storage
> 📌 *Authoritative source:* [EC2 User Guide](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/) · [EBS User Guide](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/AmazonEBS.html) — verified against AWS docs.

EC2 is elastic virtual compute. For SAA you must know **instance/purchasing options, EBS volumes, AMI sharing, and placement/grouping** — these appear constantly.

## 1. Instance purchasing options (exam core)

| Option | Use case | Commitment |
|--------|----------|------------|
| **On-Demand** | short, unpredictable, dev | none, per-second |
| **Reserved (RI)** | steady-state, 1–3 yr | up to 72% off |
| **Savings Plans** | flexible compute spend | 1–3 yr, usage-based |
| **Spot** | fault-tolerant, batch, big data | up to 90% off, can be interrupted |
| **Dedicated Host/Instance** | licensing/regulatory isolation | physical server |

> 🧠 **Exam tip:** "Flexible, cheapest for fault-tolerant batch" → **Spot**. "Steady 24/7 for 3 years" → **Reserved/Savings Plans**. "Must be physically isolated" → **Dedicated Host**.

## 2. Instance lifecycle

`pending → running → stopping → stopped → terminated`. You pay for `running` only (plus EBS/storage). `stop` keeps the root volume; `terminate` deletes it (unless `DeleteOnTermination=false`).

## 3. AMIs

An AMI is a template (OS + software + volume snapshot). AMIs are **regional**; you can copy them across regions and share/private/public. Launch permissions: `public | explicit | implicit`.

## 4. Storage: EBS vs Instance Store

| | EBS | Instance Store |
|---|-----|----------------|
| Persistence | Survives stop/terminate | Lost on stop/terminate |
| Network-attached | Yes (block) | Physical to host |
| Use | Boot + data volumes | Temp/high-IOPS cache |

EBS types: `gp3/gp2` (general SSD), `io1/io2` (provisioned IOPS, critical DB), `st1` (throughput HDD), `sc1` (cold HDD). **io2 Block Express** for sub-ms, 64k IOPS.

## 5. Security groups & placement

- **SG** = stateful firewall (allow rules only; return traffic auto-allowed).
- **Placement group:** *cluster* (low latency, single AZ), *spread* (isolate across HW), *partition* (isolate groups of instances).

## 6. Launch (CLI)

<ExampleBox title="Launch t3.micro + key + SG" lang="bash">

```bash
aws ec2 create-key-pair --key-name mykey --query 'KeyMaterial' --output text > mykey.pem
chmod 400 mykey.pem
aws ec2 create-security-group --group-name web-sg --description "web"
aws ec2 authorize-security-group-ingress --group-name web-sg --protocol tcp --port 22 --cidr 0.0.0.0/0
aws ec2 authorize-security-group-ingress --group-name web-sg --protocol tcp --port 80 --cidr 0.0.0.0/0
aws ec2 run-instances --image-id ami-0c7217cdde317cfec --instance-type t3.micro \
  --key-name mykey --security-groups web-sg --count 1
```

</ExampleBox>

(Use a current AMI ID for your region.)

</ExerciseBox>
