# Lesson 3 — EC2: Your First Virtual Server

EC2 (Elastic Compute Cloud) gives you virtual servers (instances) in the cloud. This is the foundation of most AWS architectures.

## Key concepts

- **AMI** — the image (OS + software) used to boot the instance.
- **Instance type** — CPU/RAM sizing (e.g. `t3.micro`, free-tier eligible).
- **Security group** — a virtual firewall controlling inbound/outbound traffic.
- **Key pair** — SSH credentials to log in.

## Launch an instance (CLI)

<ExampleBox title="Launch a t3.micro with a security group" lang="bash">

```bash
# 1. create a key pair
aws ec2 create-key-pair --key-name mykey --query 'KeyMaterial' \
  --output text > mykey.pem && chmod 400 mykey.pem

# 2. create a security group allowing SSH + HTTP
aws ec2 create-security-group --group-name web-sg \
  --description "web traffic"
aws ec2 authorize-security-group-ingress --group-name web-sg \
  --protocol tcp --port 22 --cidr 0.0.0.0/0
aws ec2 authorize-security-group-ingress --group-name web-sg \
  --protocol tcp --port 80 --cidr 0.0.0.0/0

# 3. launch
aws ec2 run-instances --image-id ami-0c7217cdde317cfec \
  --instance-type t3.micro --key-name mykey \
  --security-groups web-sg --count 1
```

</ExampleBox>

> Replace the AMI ID with a current one for your region (find it in the console or `aws ssm get-parameters ...`).

## Connect

<ExampleBox title="SSH in" lang="bash">

```bash
ssh -i mykey.pem ec2-user@<public-ip>
```

</ExampleBox>

## Manage instances

<ExampleBox title="Stop / start / terminate" lang="bash">

```bash
aws ec2 stop-instances --instance-ids i-0abcd1234
aws ec2 start-instances --instance-ids i-0abcd1234
aws ec2 terminate-instances --instance-ids i-0abcd1234   # irreversible!
```

</ExampleBox>

<ExerciseBox title="Launch + clean up" difficulty="Easy">
Launch a `t3.micro` instance, confirm its state is `running` via `aws ec2 describe-instances`, then **terminate** it so you are not billed.
</ExerciseBox>

<ExerciseBox title="Custom security group" difficulty="Medium">
Create a security group that allows HTTP (80) from anywhere but SSH (22) only from your own IP (use `curl https://checkip.amazonaws.com` to find it). Launch an instance into it.
</ExerciseBox>
